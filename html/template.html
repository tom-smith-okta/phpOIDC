Okta OIDC

Okta OIDC session manager

single redirect_uri

This site shows how you can manage authentication and user sessions using Okta's OIDC flows.

The site is using a single redirect_uri (redirect.php), and passing a value in the "state" parameter to indicate where a user should land after authentication.

If you are not authenticated, and you try to access page 1 or page 2, you will be redirected to authenticate.

The site also supports IDP-init flows as well (log out and click on the Okta tenant name above to test.)
